
### **产品名称构想**

*   **VoiceFlow** / **IntelliType** / **智能语音输入助手**

### **产品定位**

一款面向 Windows 平台的高级桌面端语音输入工具。它不仅提供基础的语音转文字功能，更通过集成大型语言模型（LLM）实现文本的智能修正与优化，旨在为文字工作者、程序员、内容创作者等用户提供一个**高度可定制、高精度、高效率**的语音输入解决方案。

### **核心用户**

*   需要大量文字输入，希望解放双手的用户（如作家、记者、律师）。
*   在特定场景下不方便打字的用户（如整理会议纪要、记录瞬间灵感）。
*   对语音输入准确性和流畅性有更高要求的专业人士。
*   希望利用 AI 能力优化初稿文本的创作者。

### **功能模块规划**

#### **1. 核心输入模块：快捷键驱动的无缝体验**

*   **全局快捷键触发**：
    *   **开始录音**：用户可以自定义一个全局快捷键（如 `Alt + Q`）。在任何程序中按下此快捷键，即可立即开始录音，并有清晰的视觉或听觉提示（如托盘图标变化、提示音）。
    *   **结束录音并处理**：用户按下另一个自定义的全局快捷键（或同一起始键），结束录音。程序自动进入后台处理流程。
*   **光标处自动输出**：
    *   处理完成后，最终生成的文本会像键盘输入一样，自动粘贴并输出到当前激活窗口的光标所在位置，实现真正的“语音输入法”效果。

#### **2. 智能转录引擎：双核驱动**

这是产品的核心，由两个可配置的部分组成。

*   **模块一：语音转文本 (STT - Speech-to-Text)**
    *   **多服务商支持**：内置支持多个主流的语音转文字 API 提供方（如 OpenAI Whisper, Google Speech-to-Text, 阿里云，讯飞等）。
    *   **可配置接口**：用户可以在配置中心选择当前使用的服务商，并填入自己的 API Key 和其他必要参数（如地区节点）。这给予了用户充分的选择权，可以根据价格、识别准确率来灵活切换。
*   **模块二：大模型文本后处理 (LLM Post-processing)**
    *   **功能开关**：用户可以一键开启或关闭此功能。关闭时，软件仅执行语音转文字；开启时，则会执行后续的智能修正。
    *   **大模型配置**：
        *   **Base URL**：支持配置自定义的 API 地址，兼容主流 LLM 服务商及私有化部署的模型。
        *   **API Key**：填入对应大模型的密钥。
        *   **模型选择**：允许用户指定要使用的具体模型名称（如 `gpt-4o`, `claude-3-opus-20240229`, `moonshot-v1-8k` 等）。
    *   **智能修正与 Prompt 配置**：
        *   **Prompt 管理**：用户可以预设和管理多个 Prompt (提示词) 模板。
        *   **Prompt 选择**：在执行任务时，可以选择使用哪个 Prompt。
        *   **工作原理**：软件会将 STT 生成的原始文本，结合用户选择的 Prompt，一起发送给大模型。例如，Prompt 可以是：
            > "你是一个专业的速记员和文本润色专家。以下是一段由语音直接转换的文本，可能包含口语化表达、错别字和不通顺的句子。请根据上下文，修正文本中的错误，使其更加书面化、流畅和专业，但要保持原意。这是原始文本：[此处插入STT文本]"

#### **3. 历史记录与管理中心**

这是一个非常重要的功能，方便用户追溯和复用。

*   **结构化记录**：每一次语音输入都会生成一条历史记录。
*   **记录三要素**：每条记录都清晰地包含三个部分，方便对比和检查：
    1.  **原始音频文件**：可以点击播放，回顾当时说了什么。
    2.  **原始转录文本**：由 STT API 直接生成的未经修改的文本。
    3.  **最终修正文本**：经过大模型根据 Prompt 修正后的最终文本。
*   **管理功能**：
    *   **查看与回放**：浏览所有历史记录，播放音频。
    *   **复制**：可以方便地一键复制“原始文本”或“最终文本”。
    *   **删除**：管理历史记录，删除不需要的条目。

#### **4. 配置中心**

所有可定制化的选项都集中在此处，是产品的灵魂。

*   **通用设置**：
    *   开机自启。
    *   快捷键自定义设置（开始/结束录音）。
*   **语音转文本 (STT) 配置**：
    *   下拉菜单选择 API 服务商。
    *   对应服务商的 API Key 及其他参数输入框。
*   **大模型 (LLM) 配置**：
    *   全局开关：是否启用大模型后处理。
    *   API Base URL, API Key, 模型名称输入框。
    *   Prompt 管理界面：可以新增、编辑、删除和选择默认的 Prompt 模板。
*   **历史记录设置**：
    *   设置历史记录的保存上限（如最近100条）。
    *   清理全部历史记录的按钮。

### **总结一下您想实现的产品**

您想打造的不是一个简单的语音听写工具，而是一个**“工作流自动化”和“AI 赋能”的智能语音输入平台**。

它的核心优势在于：

1.  **高度开放与可定制**：用户不仅可以选择自己信赖的 STT 服务，还能接入任何兼容的大模型，并用自定义的 Prompt 指挥 AI，让输出的文本完全符合自己的需求。
2.  **智能修正能力**：它解决了传统语音输入“仅能识别，不能理解”的痛点。通过大模型，它可以自动完成从“口语”到“书面语”的转换，修正错误，甚至根据 Prompt 进行风格润色。
3.  **完整的工作追溯**：保留音频、初稿、定稿三份数据，让每一次转换都有据可查，方便校对和复盘。
4.  **无缝的桌面端集成**：通过全局快捷键和光标处自动输出，它能完美融入用户现有的任何工作环境，真正做到“即说即得”，提升效率。
